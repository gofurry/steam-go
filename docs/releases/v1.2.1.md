# steam-go v1.2.1

`steam-go v1.2.1` is a repository hygiene and release trust update.

This release does not expand endpoint coverage or change the public SDK surface. It focuses on security guidance, contribution workflow, CI repeatability, and release readiness.

## Highlights

- added a top-level `SECURITY.md`
- added a top-level `CONTRIBUTING.md`
- added GitHub issue templates for bug reports, feature requests, and questions
- added a pull request template with compatibility, validation, documentation, and secret-safety checks
- expanded `.gitignore` for local credentials, editor metadata, logs, coverage, and build artifacts
- pinned mainline CI versions for `staticcheck` and `govulncheck`
- added a scheduled latest-toolchain advisory workflow
- added `docs/releases/checklist.md`
- upgraded `golang.org/x/net` to a version with current `govulncheck` fixes

## Compatibility

There are no intentional public API changes in this release.

The release keeps the existing `v1` boundaries:

- official Steam Web API access remains under `client.API.*`
- read-only unofficial web JSON helpers remain under `client.Web.*`
- addon behavior remains explicit and documented separately from the core SDK
- credentials, cookies, and proxy secrets should not be logged or pasted into public issues

## Security Notes

`SECURITY.md` now documents sensitive values that must not appear in issues, pull requests, logs, traces, screenshots, or examples.

This includes:

- API keys and publisher keys
- `access_token`
- `refresh_token`
- `steamLoginSecure`
- `sessionid`
- raw `Cookie` headers
- proxy URLs with userinfo
- redirect URLs containing credentials

Use `steam.RedactSensitiveURL(...)` before logging URLs that may contain Steam credentials.

## CI Notes

Mainline CI now uses pinned analyzer versions:

- `staticcheck v0.7.0`
- `govulncheck v1.3.0`

A separate scheduled workflow runs latest Go, latest `staticcheck`, and latest `govulncheck` as an advisory signal. It is not part of the normal pull request trigger.

The Go 1.26 CI lane is pinned to Go 1.26.4 so the release gate uses a standard library version with the current `govulncheck` fixes available at the time of this update.

## Validation

The intended release gate is:

- `go test ./...`
- `go test -race ./...`
- `go vet ./...`
- `staticcheck ./...`
- `govulncheck ./...`

See `docs/releases/checklist.md` before publishing the tag.

## Deferred

The following work remains intentionally deferred to later `v1.2.x` or `v1.3.x` releases:

- README adoption rewrite
- cookbook documentation
- pkg.go.dev examples
- API diff automation
- credential safety deep-dive documentation
- fixture corpus and live smoke reporting
- doctor command
