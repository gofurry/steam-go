# steam-go v1.3.0

`steam-go v1.3.0` is a maintenance automation and adoption helper release.

This release adds local maintenance tooling, fixture and smoke baselines, a diagnostic example, high-value read-only helpers, and a lightweight request observer. It does not introduce breaking changes.

## Highlights

- added `internal/tools/steamapi-sync` to generate official Steam Web API inventory, SDK coverage, and coverage diff reports
- added a scheduled coverage drift workflow that opens or updates a GitHub issue when upstream API inventory changes
- added fixture and golden regression coverage for official API decode, volatile Web payloads, redaction, and asset URL generation
- added opt-in live smoke tests under `examples/live`, skipped unless `STEAM_GO_LIVE=1` is set
- added `examples/doctor` for diagnosing official API, Storefront, Community, Market, proxy, timeout, and credential configuration problems
- added read-only helper APIs for reviews pagination, inventory pagination, batch app details, and batch market price overview
- added `WithRequestObserver(...)` with sanitized `RequestEvent` values for lightweight request visibility
- expanded English and Chinese docs, cookbook pages, Web reference, and release checklist coverage

## Compatibility

This release only adds compatible public APIs.

Existing endpoint behavior, request controls, addon behavior, and `v1` package boundaries remain unchanged. New helper APIs continue to use the existing client request layer, including context cancellation, timeout, retry, rate limit, traffic policy, proxy, body cap, cookie jar, and error handling behavior.

## Security and Safety

- Doctor, live smoke, examples, and request observer output avoid printing API keys, access tokens, cookies, proxy passwords, raw query strings, headers, and bodies.
- Web helper additions remain read-only and do not add browser automation, login automation, purchase, sale, trade, or bulk account automation behavior.
- Coverage automation reports endpoint metadata only; it does not require or store credentials.

## Known Limitations

- `steamapi-sync` is an internal maintenance tool, not a stable user CLI.
- Coverage drift issues are advisory and do not automatically update code or open pull requests.
- `examples/doctor` is a diagnostic example, not a stable CLI product.
- Live smoke tests remain opt-in because they depend on network access and Steam availability.
- Unofficial Web payloads remain volatile; stable outer fields are preferred over aggressively typed nested payloads.
