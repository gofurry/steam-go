# v1.3.2 Release Notes

v1.3.2 is a small compatible feature release focused on Steam Store news/event collection and safe downstream content handling.

## Highlights

- Storefront now exposes the public adjacent partner events endpoint used by Steam event/news pages.
- A new `addons/markup` package converts common Steam BBCode, sanitizes HTML, and produces plain-text summaries for indexing or metadata.

## Added

- Added `client.Web.Storefront.GetAdjacentPartnerEvents` and `GetAdjacentPartnerEventsRaw` for `store.steampowered.com/events/ajaxgetadjacentpartnerevents`.
- Added typed Store events response models with raw payload preservation for volatile nested fields.
- Added `addons/markup` helpers:
  - `SteamBBCodeToHTML`
  - `CleanSteamContent`
  - `CleanHTML`
  - `PlainText`
  - `Summary`
- Added support for common Steam content patterns, including `{STEAM_CLAN_IMAGE}`, `[/*]` list item closures, escaped bracket text, and escaped URL attributes.
- Added `examples/markup` as a runnable markup-cleaning example.

## Changed

- Golden tests now normalize line endings before comparison so the suite remains stable across Windows and Unix checkouts.
- Documentation now includes Store events and markup helper references in English and Chinese.

## Notes

- Store events remain an unofficial web surface. The SDK exposes a stable typed subset and preserves raw nested payloads for volatile fields.
- Markup helpers sanitize by default and are intended for caller-controlled storage, indexing, or rendering pipelines.
- YouTube iframe output is disabled by default; callers can opt in with `markup.WithYouTubeEmbeds(true)`.
- This release does not add account automation, purchase, trade, sell, or browser fallback behavior.

## Compatibility

- No breaking changes.
- New APIs are additive.
- `github.com/microcosm-cc/bluemonday` is added as a direct dependency for HTML sanitization.

## Validation

- `go test ./...`
- `go run ./examples/markup`
