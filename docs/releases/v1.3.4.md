# v1.3.4 Release Notes

v1.3.4 is a stabilization and hygiene release focused on safer request behavior,
raw HTTP boundaries, observability completeness, diagnostics, and production
usage guidance.

## Highlights

- Made retry behavior method-aware so non-idempotent methods are not retried by
  default.
- Added raw HTTP host policies for exact hosts, Steam hosts, suffix matches, and
  Steam static/CDN hosts.
- Added `RequestEvent.ConditionalHit` and observer events for `304 Not Modified`
  cache revalidation.
- Improved raw HTTP retry body handling so `GetBody` is opened only per attempt.
- Improved malformed URL redaction and added `RedactSensitiveText(...)` for
  common log and diagnostic strings.
- Unified internal SteamID64 validation across selected SDK, Web, and addon
  surfaces.
- Improved `addons/assets` batch cancellation during job submission.
- Added coverage triage, production cookbooks, addon safety notes, and an
  observer example.

## Compatibility

- No breaking changes.
- New public APIs are additive.
- Behavior changes are safer defaults, internal fixes, or opt-in helpers.

## Security and Safety

- Raw HTTP remains opt-in and requires absolute URLs.
- Host policies help callers avoid unsafe raw HTTP access to untrusted URLs.
- Redaction helpers are best-effort and conservative; they do not classify every
  possible secret format.
- Request observer events remain sanitized and do not include query strings,
  headers, bodies, cookies, credentials, or proxy passwords.

## Non-goals

- No account automation expansion.
- No purchase, sale, trade, or browser fallback behavior.
- No bulk claim behavior.
- No endpoint expansion for coverage numbers.
