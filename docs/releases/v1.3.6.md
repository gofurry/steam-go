# v1.3.6 Release Notes

v1.3.6 adds official Store item asset discovery for Steam's newer hashed asset
paths while keeping the existing AppID-only static URL builders unchanged.

## Highlights

- Added `client.API.StoreBrowseService.GetItems` and `GetItemsRaw` for
  `IStoreBrowseService/GetItems/v1`.
- Added `addons/assets.FetchStoreItemAssetURLs` for official Store item asset
  discovery through StoreBrowse metadata.
- Added Store item asset kinds such as `header_2x`, `capsule_main_2x`,
  `capsule_small_2x`, `hero_capsule_2x`, `library_hero_2x`,
  `page_background`, and `page_background_raw`.
- Added `Digest`, `Filename`, and `Source` metadata to asset URL, verify, read,
  and download results.
- Added `VerifyStoreItemAssets`, `ReadStoreItemAssets`, and
  `DownloadStoreItemAssets` wrappers that reuse the existing assets helper
  pipeline.
- Updated the assets example and documentation for official hashed asset
  discovery.

## Compatibility

- No breaking changes.
- Existing static builders such as `HeaderURLs`, `LibraryHeroURLs`,
  `AllWithLanguage`, and `DownloadAppAssets` keep their existing behavior.
- `FetchStoreMediaURLs` remains focused on Storefront appdetails screenshots,
  movies, and backgrounds.
- New APIs are additive.

## Boundaries

- Store item asset discovery uses official Steam Web API metadata.
- It does not integrate SteamGridDB.
- It does not parse Steam client appinfo or use Steam Client / CM protocols.
- Missing asset fields are skipped instead of treated as hard failures.
