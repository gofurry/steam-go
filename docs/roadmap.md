# 路线图

这个文档用于记录 `steam-go` 的近期迭代规划。随着项目从纯 Steam Web API SDK 逐步扩展为一个也能支持部分 Steam 商店公开页面接入的工具包，我们会更关注稳定性、可维护性，以及面向公开页面流量场景的抗风控能力。

## v1.0.0-alpha-1

- 建立根级 `Client` 与 `client.API.*` 分组服务结构
- 保持请求执行层轻量、可复用
- 支持 API key、access token、timeout、retry、rate limit、proxy 注入
- 将 addon 能力与核心 Web API 客户端解耦

## v1.0.0-alpha-3

重点方向：为后续接入部分 Steam 商店公开页面做稳定性与抗风控准备。这类流量更接近浏览器访问行为，而不是官方开放 Web API 的标准调用模式。

### 计划中的代理与路由能力

- 代理健康检查与被动失败评分
- 对不健康代理进行临时冷却或类似熔断的摘除
- 支持混合质量代理池的加权轮转
- 支持会话粘性路由，让同一会话在需要时固定走同一个代理
- 按 API host、商店 host、OpenID 流程、公开页面 path 做路由级代理选择
- 暴露每个代理的成功率、延迟、重试次数、拦截率等观测指标

### 计划中的凭证与请求画像能力

- 官方 API 流量继续保持简单的 API key / access token 轮转
- 面向商店公开页面，增加浏览器式请求画像轮转能力
- 提供可选的 header profile 预设，而不是零散地随机化单个 header
- 提供可选的 `Referer` 策略支持，包括固定、路由式、上下文生成式
- 支持面向公开页面流程的会话级 cookie jar

### 计划中的抗风控与稳定性能力

- 按 host 和会话维度自适应限速
- 针对高批量列表/详情抓取的有界并发控制
- 针对公开页面优化过的带抖动重试与冷却策略
- 在上游支持的前提下使用 `ETag` / `If-Modified-Since` 条件请求
- 对重复公开页面请求增加短时缓存
- 为未来可能接入的自定义 TLS / 浏览器回退抓取策略预留 transport 扩展点
- 结构化识别常见的 403 / 429 / 重定向 / 挑战页模式
- 提供指标与调试 hook，方便区分网络失败、上游限流、风控拦截

### alpha-3 明确不做的事情

- 不内置重量级通用爬虫框架
- 不内置挑战页绕过逻辑
- 不引入不透明的“魔法默认行为”去干扰官方 API 调用语义
- 不把整个 SDK 绑定到单一反爬服务商或浏览器自动化方案上
