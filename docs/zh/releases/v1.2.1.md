# steam-go v1.2.1

`steam-go v1.2.1` 是一次仓库卫生和发布可信度更新。

这个版本不扩展 endpoint 覆盖，也不改变公开 SDK API。重点是补齐安全说明、贡献流程、CI 可重复性和发布检查流程。

## 重点

- 新增根目录 `SECURITY.md`
- 新增根目录 `CONTRIBUTING.md`
- 新增 bug report、feature request、question 三类 GitHub issue template
- 新增 pull request template，覆盖兼容性、验证、文档和 secret safety 检查
- 扩展 `.gitignore`，覆盖本地凭据、编辑器元数据、日志、coverage 和构建产物
- 固定主线 CI 中的 `staticcheck` 与 `govulncheck` 版本
- 新增 latest toolchain 定时预警 workflow
- 新增 `docs/releases/checklist.md`
- 升级 `golang.org/x/net` 到当前 `govulncheck` 已修复版本

## 兼容性

本版本没有有意引入公开 API 变更。

现有 `v1` 边界保持不变：

- 官方 Steam Web API 继续放在 `client.API.*`
- 只读、非官方 Web JSON helper 继续放在 `client.Web.*`
- addon 行为继续保持显式、独立文档化
- credentials、cookies、proxy secrets 不应出现在 public issue 或日志中

## 安全说明

`SECURITY.md` 现在明确列出了不能出现在 issue、PR、日志、trace、截图或示例中的敏感值。

包括：

- API key 和 publisher key
- `access_token`
- `refresh_token`
- `steamLoginSecure`
- `sessionid`
- 原始 `Cookie` header
- 带用户名或密码的 proxy URL
- 包含凭据的 redirect URL

记录可能包含 Steam 凭据的 URL 前，应使用 `steam.RedactSensitiveURL(...)` 做脱敏。

## CI 说明

主线 CI 固定以下分析工具版本：

- `staticcheck v0.7.0`
- `govulncheck v1.3.0`

另有一个独立定时 workflow 会运行 latest Go、latest `staticcheck` 和 latest `govulncheck`，用于提前发现生态变化。它不属于普通 pull request 触发路径。

Go 1.26 CI 线固定到 Go 1.26.4，以确保发布门禁使用当前 `govulncheck` 已修复的标准库版本。

## 验证

建议发布门禁：

- `go test ./...`
- `go test -race ./...`
- `go vet ./...`
- `staticcheck ./...`
- `govulncheck ./...`

发布 tag 前请先检查 `docs/releases/checklist.md`。
