# steam-go v1.3.0

`steam-go v1.3.0` 是一次维护自动化与采用体验增强版本。

这个版本新增本地维护工具、fixture 与 smoke 基线、诊断示例、高价值只读 helper 和轻量 request observer。它不引入 breaking change。

## 重点

- 新增 `internal/tools/steamapi-sync`，用于生成官方 Steam Web API inventory、SDK coverage 和 coverage diff 报告
- 新增 scheduled coverage drift workflow，当上游 API inventory 变化时开或更新 GitHub issue
- 新增 fixture 与 golden regression，覆盖官方 API decode、高波动 Web payload、redaction 和 asset URL generation
- 新增 `examples/live` opt-in live smoke tests，默认 skip，只有设置 `STEAM_GO_LIVE=1` 才触网
- 新增 `examples/doctor`，用于诊断 official API、Storefront、Community、Market、代理、timeout 和凭据配置问题
- 新增 reviews paginator、inventory paginator、batch app details、batch market price overview 等只读 helper
- 新增 `WithRequestObserver(...)` 与脱敏 `RequestEvent`，提供轻量请求观测入口
- 同步扩展中英文 docs、cookbook、Web reference 和 release checklist

## 兼容性

本版本只新增兼容 public API。

现有 endpoint 行为、请求控制、addon 行为和 `v1` 包边界保持不变。新增 helper 继续走现有 client 请求层，因此仍然遵守 context cancellation、timeout、retry、rate limit、traffic policy、proxy、body cap、cookie jar 和 error handling 行为。

## 安全与边界

- Doctor、live smoke、examples 和 request observer 输出避免打印 API key、access token、cookie、proxy password、raw query、headers 和 bodies。
- Web helper 保持只读，不新增 browser automation、登录自动化、购买、出售、交易或批量账号自动化能力。
- Coverage automation 只报告 endpoint metadata，不需要也不会保存凭据。

## 验证

`v1.3.0` 工作在实现过程中按项目发布门禁验证：

- `go test ./...`
- `go test -race ./...`
- `go vet ./...`
- `staticcheck ./...`
- `govulncheck ./...`
- public API diff incompatible change checks

发布 tag 前请检查 [Release Checklist](../../releases/checklist.md)。

## 已知边界

- `steamapi-sync` 是内部维护工具，不是稳定用户 CLI。
- Coverage drift issue 只作为维护提醒，不会自动改代码或开 pull request。
- `examples/doctor` 是诊断示例，不是稳定 CLI 产品。
- Live smoke tests 保持 opt-in，因为它们依赖网络和 Steam 上游可用性。
- 非官方 Web payload 仍然高波动，优先稳定 outer fields，不强行 typed 化 nested payload。
