# v1.3.4 Release Notes

v1.3.4 是一次稳定性与维护性版本，重点是更安全的请求行为、raw HTTP
边界、观测完整性、诊断能力和生产使用文档。

## Highlights

- Retry 行为改为 method-aware，非幂等方法默认不自动重试。
- 新增 raw HTTP host policy，覆盖 exact host、Steam host、suffix match 和 Steam static/CDN host。
- 新增 `RequestEvent.ConditionalHit`，并为 `304 Not Modified` cache revalidation 补 observer 事件。
- 改进 raw HTTP retry body 处理，`GetBody` 只在每次 attempt 前打开。
- 改进 malformed URL 脱敏，并新增 `RedactSensitiveText(...)` 处理常见日志和诊断文本。
- 在部分 SDK、Web 和 addon surface 中统一内部 SteamID64 校验。
- 改进 `addons/assets` batch helper 在 job submission 阶段的 context cancellation。
- 新增 coverage triage、生产 cookbook、addon safety 文档和 observer 示例。

## Compatibility

- 无 breaking change。
- 新 public API 都是兼容性加法。
- 行为变化属于更安全默认值、内部修复或 opt-in helper。

## Security and Safety

- Raw HTTP 仍然是 opt-in 能力，并且只接受 absolute URL。
- Host policy 帮助调用方避免对不可信 URL 发起不安全 raw HTTP 请求。
- Redaction helper 是 best-effort 且保守的，不尝试识别所有可能的 secret 格式。
- Request observer 事件保持脱敏，不包含 query string、header、body、cookie、credential 或 proxy password。

## Non-goals

- 不扩张账号自动化。
- 不新增购买、出售、交易或 browser fallback。
- 不新增批量领取行为。
- 不为了 coverage 数字扩张 endpoint。
