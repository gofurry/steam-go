# v1.3.6 Release Notes

v1.3.6 为 Steam 较新的 hashed asset 路径新增官方 Store item asset discovery，
同时保持现有 AppID-only 静态 URL builder 行为不变。

## Highlights

- 新增 `client.API.StoreBrowseService.GetItems` 和 `GetItemsRaw`，覆盖
  `IStoreBrowseService/GetItems/v1`。
- 新增 `addons/assets.FetchStoreItemAssetURLs`，通过 StoreBrowse metadata
  发现官方 Store item asset URL。
- 新增 `header_2x`、`capsule_main_2x`、`capsule_small_2x`、
  `hero_capsule_2x`、`library_hero_2x`、`page_background`、
  `page_background_raw` 等 Store item asset kind。
- 在 URL、verify、read、download 结果中新增 `Digest`、`Filename` 和
  `Source` metadata。
- 新增 `VerifyStoreItemAssets`、`ReadStoreItemAssets`、
  `DownloadStoreItemAssets`，继续复用现有 assets helper 流水线。
- 更新 assets example 和文档，展示官方 hashed asset discovery。

## Compatibility

- 无 breaking change。
- `HeaderURLs`、`LibraryHeroURLs`、`AllWithLanguage`、`DownloadAppAssets`
  等现有静态 builder 行为不变。
- `FetchStoreMediaURLs` 仍然只负责 Storefront appdetails 中的截图、视频和背景。
- 新 public API 均为兼容性加法。

## Boundaries

- Store item asset discovery 使用官方 Steam Web API metadata。
- 不接入 SteamGridDB。
- 不解析 Steam client appinfo，也不使用 Steam Client / CM 协议。
- 缺失的 asset 字段会被跳过，不作为硬错误。

## Validation

- `go test ./...`
- `go test -race ./...`
- `go vet ./...`
- `staticcheck ./...` 在当前环境无法运行：已安装的 staticcheck binary 使用
  Go 1.22.10 构建，而本 module 要求 Go 1.25.0。
- `govulncheck ./...` 当前环境未安装。
