# 为什么需要安全默认值

`WithSafeDefaults()` 存在的原因是：外部 API 客户端不应该一上来就使用激进策略。

## 它做什么

`WithSafeDefaults()` 会应用一个保守配置：

```text
retry = 2
rate limit = 3 requests/second
burst = 3
```

## 为什么重要

SDK 新用户可能一开始并不知道：

- 代码实际调用 Steam API 的频率
- 请求是否并发发生
- 失败时是否会自动重试
- 部署后是否有多个实例
- 同一个 API key 是否被其他任务共用

安全默认值可以减少误用，并让首次接入更稳定。

## 什么时候使用

适合：

- 示例代码
- 小型服务
- CLI 工具
- 早期接入
- 会访问真实外部端点的测试

```go
client, err := steam.NewClient(
    steam.WithSafeDefaults(),
)
```

## 什么时候覆盖

当你已经具备这些条件时，可以显式覆盖：

- 队列
- metrics
- 生产流量估算
- 明确的并发控制
- 服务级别的 rate budget

```go
client, err := steam.NewClient(
    steam.WithRetry(2),
    steam.WithRateLimiter(rate.Limit(10), 10),
)
```

## 设计原则

SDK 应该让安全路径足够容易，同时允许高级用户精细调整行为。
