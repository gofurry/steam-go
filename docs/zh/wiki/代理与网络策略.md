# 代理与网络策略

`steam-go` 没有把代理设计成一个简单的固定 URL，而是抽象成 `ProxySelector`。

这样可以让代理策略更容易组合、测试，并适配不同类型的 Steam 流量。

## 为什么需要 ProxySelector

不同 Steam 流量可能需要不同网络路径：

- 官方 Web API 流量
- Steam Community OpenID 验证
- 公开 Store 页面流量
- 区域敏感请求
- 类浏览器会话流量
- 内部测试或调试流量

## 代理类型

| Selector | 使用场景 |
|---|---|
| Static Proxy | 所有请求走一个代理 |
| Round-Robin Proxy | 多个代理轮询 |
| Health-Checked Proxy | 临时避开失败代理 |
| Sticky Proxy | 同一个 session 固定代理 |
| Routing Proxy | 按 host/path 路由 |

## 固定代理

```go
selector, err := steam.NewStaticProxySelector("http://127.0.0.1:7897")

client, err := steam.NewClient(
    steam.WithProxySelector(selector),
)
```

## 轮询代理

```go
selector, err := steam.NewRoundRobinProxySelector(
    "http://127.0.0.1:7897",
    "http://127.0.0.1:7898",
)
```

## 健康检查代理

```go
selector, err := steam.NewHealthCheckedRoundRobinProxySelector(
    steam.DefaultProxyHealthConfig(),
    "http://127.0.0.1:7897",
    "http://127.0.0.1:7898",
)

metrics := selector.(steam.ProxyMetricsProvider).ProxyMetricsSnapshot()
```

适合代理可用性不稳定的场景。

## 粘性代理

```go
base, _ := steam.NewRoundRobinProxySelector(
    "http://127.0.0.1:7897",
    "http://127.0.0.1:7898",
)

selector := steam.NewStickyProxySelector(base)

ctx := steam.WithProxySessionKey(context.Background(), "session-1")
```

适合类浏览器或类登录流程需要保持同一代理出口的场景。

## 路由代理

```go
selector, err := steam.NewRoutingProxySelector(
    steam.ProxyRoute{
        Host:       "api.steampowered.com",
        PathPrefix: "/ISteamUser/",
        ProxyURL:   "http://127.0.0.1:7897",
    },
    steam.ProxyRoute{
        Host:       "steamcommunity.com",
        PathPrefix: "/openid/",
        ProxyURL:   "",
    },
)
```

`ProxyURL` 为空表示直连。

## 实用建议

- 优先使用一个统一代理策略，而不是在各处散落自定义 transport。
- 只有在需要会话亲和时才使用 sticky proxy。
- 代理可用性不稳定时使用 health checking。
- 官方 API、OpenID、Store 页面需要不同网络路径时使用 routing。
- 不要用无限重试掩盖网络失败。
