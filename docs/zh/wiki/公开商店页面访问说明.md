# 公开商店页面访问说明

公开 Store 页面和官方 Steam Web API 端点不是一回事。

它们可以有价值，但应该被视为类浏览器的公开网页流量，而不是稳定 typed API 流量。

从 `v1.1.0` 开始，`steam-go` 也提供了一小层只读的 `client.Web.*`，用于接入少量高价值的 Storefront、Community 与 Market JSON 接口。这些方法本身是稳定的 Go API，但它们的上游 payload 仍应被视为非官方或高波动的 Web surface。

现在 `client.Web.*` 也包含面向常见只读流程的 handler 翻页和小型批量 helper：

- `client.Web.Storefront.ListAppReviews`
- `client.Web.Community.ListInventory`
- `client.Web.Storefront.GetAppDetailsBatch`
- `client.Web.Market.GetPriceOverviewBatch`

这些 helper 会复用底层单项方法的请求控制，并保持只读边界。

## 官方 API 与公开 Store 页面

| 官方 Web API | 公开 Store 页面 |
|---|---|
| 偏 JSON | 偏 HTML |
| 可能需要 API key | 可能需要类浏览器请求 |
| 更适合 typed SDK 方法 | 更适合 best-effort 提取 |
| 请求结构更稳定 | 页面结构可能变化 |
| 更像后端 API | 更像公开网站行为 |

## 推荐 SDK 能力

访问 Store 页面类流量时，可以考虑：

- `TrafficClassPublicStorePage`
- `TrafficClassCommunityWeb`
- `TrafficClassMarketWeb`
- `HeaderProfile`
- `RefererSelector`
- `TrafficCachePolicy`
- `TrafficBlockPolicy`
- `HostControl`
- `SessionControl`
- 特定路由代理策略

## Header Profile

```go
profile := steam.DefaultPublicStoreHeaderProfileEN()
```

或：

```go
profile := steam.DefaultPublicStoreHeaderProfileZH()
```

## Referer

```go
referer, err := steam.NewStaticRefererSelector(
    "https://store.steampowered.com/search/",
)
```

## 短缓存

```go
Cache: &steam.TrafficCachePolicy{
    TTL: time.Minute,
}
```

## 拦截检测

```go
BlockPolicy: &steam.TrafficBlockPolicy{
    HTMLSniffBytes: 4096,
}
```

## 实用建议

- 如果 SDK 已经直接暴露了需要的 JSON 接口，优先使用内置的 `client.Web.Storefront.*`、`client.Web.Community.*`、`client.Web.Market.*` 方法。
- 对 reviews、inventory、app details、market price overview 等常见循环，优先使用 paginator 或 batch helper，而不是在业务代码里手写重复请求。
- 不要把 HTML 页面当作稳定 API。
- 页面解析代码应和核心 Web API client 隔离。
- 重复读取应使用缓存。
- 使用低并发。
- 注意 `403`、`429` 和 HTML challenge 页面。
- 如果有官方 Web API 方法，优先使用官方接口。
