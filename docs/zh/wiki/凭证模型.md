# 凭证模型

`steam-go` 把不同类型的 Steam 凭证拆开处理，而不是全部当作一个通用 token。

这样做更安全，也更容易理解边界。

## API Key

Steam Web API key 用于需要 `key` 参数的接口。

```go
client, err := steam.NewClient(
    steam.WithAPIKey("your-key"),
)
```

## 多 API Key 轮换

```go
client, err := steam.NewClient(
    steam.WithAPIKeys("key-a", "key-b", "key-c"),
)
```

它适合在应用合法拥有多个 key 的前提下分摊请求压力。

## 健康检查 API Key

```go
client, err := steam.NewClient(
    steam.WithHealthCheckedAPIKeys(
        steam.DefaultAPIKeyHealthConfig(),
        "key-a",
        "key-b",
    ),
)
```

当某个 key 连续遇到 `401` 或 `429` 时，可以临时进入 cooldown。

## Access Token

Access token 和 Web API key 是不同的凭证。

```go
client, err := steam.NewClient(
    steam.WithAccessToken("access-token"),
)
```

只应该在需要 token-based access 的接口或流程中使用。

## OpenID

OpenID 用于浏览器中的 Steam 登录。

它验证用户的 Steam 身份，并返回 SteamID64。

OpenID 不替代：

- Steam Web API key
- access token
- 应用自己的 session
- 用户资料获取接口

## 凭证应保留在后端

不要把 API key 或 access token 暴露给前端 JavaScript。

推荐模式：

```text
Browser -> Your Backend -> steam-go -> Steam
```

避免：

```text
Browser -> Steam with raw API key
```

## 日志安全

Steam 凭证经常出现在 query 参数里。

打印 URL 前应先脱敏：

```go
safeURL := steam.RedactSensitiveURL(rawURL)
```

## 实用规则

用能完成任务的最小凭证：

| 需求 | 凭证 |
|---|---|
| 需要 key 的公开 Web API 数据 | API Key |
| 用户授权的受保护操作 | Access Token |
| 浏览器登录身份确认 | OpenID |
| 服务端发行商操作 | Publisher Web API Key |
