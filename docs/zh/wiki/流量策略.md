# 流量策略

`TrafficPolicy` 用于让不同类型的请求使用不同的网络行为。

这是 `steam-go` 中最重要的工程抽象之一。

## 流量分类

| Class | 含义 |
|---|---|
| `TrafficClassOfficialAPI` | 普通 typed Steam Web API 流量 |
| `TrafficClassPublicStorePage` | Storefront Web JSON 或类浏览器 Store 流量 |
| `TrafficClassCommunityWeb` | Steam Community 库存类 Web JSON 流量 |
| `TrafficClassMarketWeb` | Steam Community Market Web JSON 流量 |

## 为什么需要它

官方 API 请求和公开 Store 页面请求不应该总是共用同一套策略。

官方 API 流量通常关心：

- API key
- retry
- rate limit
- response decoding
- typed responses

公开 Store 页面流量可能关心：

- 浏览器请求头
- cookies
- Referer
- block detection
- short cache
- 更低并发
- proxy routing

Community 或 Market Web JSON 流量通常也会额外关心：

- 与官方 API 请求隔离的限流策略
- cookie jar 复用
- 重复读取时的短缓存
- 遇到 `403`、`429` 或 challenge HTML 时的 block detection

## 基础示例

```go
client, err := steam.NewClient(
    steam.WithTrafficPolicy(
        steam.TrafficClassPublicStorePage,
        steam.TrafficPolicy{
            RateLimiter: &steam.TrafficRateLimiterPolicy{
                Limit: 10,
                Burst: 10,
            },
        },
    ),
)
```

## Store 页面策略示例

```go
profile := steam.DefaultPublicStoreHeaderProfileEN()

client, err := steam.NewClient(
    steam.WithTrafficPolicy(
        steam.TrafficClassPublicStorePage,
        steam.TrafficPolicy{
            HeaderProfile: &profile,
            Cache: &steam.TrafficCachePolicy{
                TTL: time.Minute,
            },
            BlockPolicy: &steam.TrafficBlockPolicy{
                HTMLSniffBytes: 4096,
            },
        },
    ),
)
```

## 单个请求指定流量类型

```go
ctx := steam.WithTrafficClass(
    context.Background(),
    steam.TrafficClassPublicStorePage,
)
```

在日常使用里，内置的 `client.Web.*` 方法已经会自动选择自己的默认 traffic class：

- `client.Web.Storefront.*` -> `TrafficClassPublicStorePage`
- `client.Web.Community.*` -> `TrafficClassCommunityWeb`
- `client.Web.Market.*` -> `TrafficClassMarketWeb`

## 可覆盖策略

`TrafficPolicy` 可以覆盖：

- proxy selector
- cookie jar
- rate limiter
- retry policy
- host control
- session control
- short cache
- block detection
- header profile
- Referer selector
- transport hook

## Request Observer

需要轻量请求元数据用于日志或指标，又不想引入 OpenTelemetry 依赖时，可以使用 `WithRequestObserver(...)`：

```go
client, err := steam.NewClient(
    steam.WithRequestObserver(steam.RequestObserverFunc(func(event steam.RequestEvent) {
        _ = event.TrafficClass
        _ = event.Path
        _ = event.StatusCode
        _ = event.Duration
    })),
)
```

Observer event 会在 SDK 请求完成后发出，包含 traffic class、method、host、不带 raw query 的 path、status code、error kind、retry attempts、cache hit、block detection 和 duration。事件不包含 raw query、header、body、API key、token、cookie 或 proxy 密码。

## 设计原则

流量行为应尽量靠近 SDK 边界。

不要把限流、Cookie、代理选择、浏览器请求头散落到业务代码中。
