# 错误处理

`steam-go` 使用 `*steam.APIError` 对 SDK 错误进行分类。

目标是让调用方不用解析错误字符串，也能判断失败阶段。

## 错误类型

| Kind | 含义 |
|---|---|
| `request_build` | 构造请求失败 |
| `transport` | 网络或传输层失败 |
| `http_status` | 非成功 HTTP 状态码 |
| `decode` | 响应解码失败 |
| `api_response` | Steam API 返回应用层错误 |

## 示例

```go
var apiErr *steam.APIError
if errors.As(err, &apiErr) {
    fmt.Println(apiErr.Kind)
    fmt.Println(apiErr.StatusCode)
    fmt.Println(apiErr.BodyPreview)
}
```

## 应该记录什么

推荐记录：

- error kind
- HTTP status code
- request method
- 脱敏后的 URL
- 简短 body preview
- retry count
- traffic class

## 不应该记录什么

避免记录：

- 原始 API key
- 原始 access token
- 未脱敏的完整 URL
- 可能包含敏感信息或过大的完整响应体

## URL 脱敏

```go
safeURL := steam.RedactSensitiveURL(rawURL)
```

## 推荐处理方式

| Error Kind | 建议处理 |
|---|---|
| `request_build` | 修复调用参数或 SDK bug |
| `transport` | 安全情况下重试 |
| `http_status` | 检查状态码、限流、认证或拦截 |
| `decode` | 检查 raw response 或更新模型 |
| `api_response` | 按 Steam 应用层错误语义处理 |
